## JUnit
#### JUnit 또한 프레임워크로 프레임워크가 코드를 제어할 수 있게 룰을 따라줘야 한다

<br>

### 테스트 메소드 전환
* `public`으로 접근자 선언
*  `void`로 리턴값 선언
*  0개 인풋 파라미터
* `@Test` 에노테이션 붙이기

<br>

### 검증 코드 전환
* static method
  * import static org.hamcrest.CoreMatchers.is;
  * import static org.junit.Assert.assertThat; 

<br>

### 테스트 결과의 일관성
* CRUD의 CU성 작업이 테스트 과정에 있다면, 테스트 종료 이후에 모두 D해줘서, 테스트 결과의 일관성을 유지한다
  * deleteAll()
  * getCount()
