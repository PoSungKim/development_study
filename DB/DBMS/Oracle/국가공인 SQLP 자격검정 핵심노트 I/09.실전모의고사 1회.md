### OR 조건 - 인덱스 존재 && 인덱스 미존재
* 예시)
  ```sql
  select *
  from table
  where a='a' or b='b';
  -- 인덱스 존재하는 조건 a
  -- 인덱스 미존재하는 조건 b
  ```
* 결과
  * Index Full Scan

<br>

### Index Range Scan
* 인덱스 사용 불가능한 케이스
  * 인덱스 선두칼럼이 숫자형이라서 자동 형변환이 발생할 때
* 아쉬운 설계 케이스
  * 값이 들어올 수도 있고 안 들어올 수 있을 때 LIKE 처리하기
    * 값이 안 들어왔을 때 Index Full Scan이 발생한다
